## 프로젝트

### AI 기반 지식 검색·챗봇 플랫폼 (KB QnA)

**써로마인드** · 2025.12 ~ 진행 중 · 기여도 30%

**Tech** · Next.js 15, React 19, TypeScript, Zustand, TanStack Query, Radix UI, Tailwind CSS 4, Biome, Playwright, Sentry

기존 검색 시스템이 키워드 중심이라 문맥 이해가 부족하고, 문서가 여러 저장소에 분산되어 있어 사용자가 필요한 정보를 빠르게 찾기 어려운 문제가 있었습니다. 이를 해결하기 위해 RAG 기반 검색 흐름과 개인화 KB 구조를 프론트엔드에 자연스럽게 포함할 수 있는 아키텍처를 설계했습니다.

가장 먼저 해결해야 했던 것은 **실시간 스트리밍의 안정성**이었습니다. AI 응답은 토큰 단위로 수신되는데, 메인 스레드에서 SSE 청크를 파싱하면 긴 응답 중 UI가 버벅이는 문제가 발생했습니다. 이를 해결하기 위해 별도 Web Worker 스레드에서 SSE 파싱을 처리하고 postMessage로 상태를 업데이트하는 구조를 설계했습니다. 네트워크 불안정 상황에 대비하여 ExponentialBackoff 재시도 로직과 오프라인 큐도 함께 구현해, 연결이 끊겨도 사용자 경험이 끊기지 않도록 했습니다.

다음으로 **검색 결과의 신뢰성** 문제가 있었습니다. 엔터프라이즈 환경에서는 "AI가 어디서 이 답변을 가져왔는지"를 투명하게 보여주는 것이 중요했습니다. 검색 결과를 단순 답변이 아니라 출처·근거 스니펫을 포함한 Explainable UI로 제공하도록 설계하여 신뢰성을 확보했습니다.

또한 사용자가 업로드하는 문서의 **포맷이 다양**했기 때문에(PDF, DOCX, CSV, TXT), 각 포맷에 최적화된 뷰어를 Dynamic Import로 구성했습니다. PDF는 lazy loading과 가상 스크롤로 대용량 문서에 대응하고, DOCX는 HTML 변환, CSV는 인코딩 폴백 처리를 적용했습니다.

프로젝트 전체에는 FSD(Feature-Sliced Design) 아키텍처를 도입하고 Steiger로 레이어 간 의존성 규칙을 자동 검증하도록 했습니다. 테스트 전략은 Vitest(단위) + Testing Library(컴포넌트) + Playwright(E2E) 3단계 체계로 구축하여 품질을 보장하고 있습니다.

**써로마인드 (삼성 E&A 외주)** · 2025.06 ~ 진행 중 · 기여도 40%

먼저 AI가 추출한 심볼·라인 데이터를 웹에서 직접 다룰 수 있도록 **Konva 기반 시각화·편집 아키텍처**를 설계했습니다. 그룹핑·추가·삭제·편집·멀티 선택 등 복잡한 편집 기능을 구현하되, 폐쇄망 환경에서도 안정적인 렌더링이 가능하도록 번들 최적화·리소스 로컬 호스팅·대용량 도면 가상화 전략을 적용해 성능 문제를 해결했습니다.

또 하나의 도전은 **커뮤니케이션**이었습니다. 외주사·기획자·AI팀 간 해석 차이와 요구사항 변경이 반복되었기 때문에, 프론트엔드 담당자로서 도면 규칙, 심볼 정의, 편집 제약을 기술적으로 정리하여 명세를 재구축했습니다. 불규칙한 레거시 API 스펙에는 Zod 기반 요청/응답 스키마 검증 레이어를 구축하여 대응했습니다. 이를 통해 기능 스코프가 명확해지고, AI 분석 → 시각화 → 사용자 편집 → 저장까지 이어지는 전 과정을 사내망 환경에서 안정적으로 수행할 수 있는 구조를 확립했습니다.

이를 해결하기 위해 Express.js 기반의 SSG 전환을 통해 주요 랜딩 페이지를 사전 렌더링해 SEO 점수를 개선했습니다. 초기 로딩 속도는 코드 스플리팅, 이미지 리사이즈, WebP 변환을 통해 최적화했습니다. Kakao Map API와 공공 데이터 API를 병합하여 부지 경계·용도 지역 등 다각형 편집 기능을 구현했고, Three.js를 사용해 주택 타입별 3D 모델을 로드하여 실시간 견적/조감도 동기화 기능을 개발했습니다.
